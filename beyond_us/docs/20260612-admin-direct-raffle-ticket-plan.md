# 2026-06-12 관리자 추첨권 직접 지급 계획

## 목표

운영자가 앱 가입자 탭에서 유저를 검색한 뒤 추첨권 1장을 바로 지급할 수 있게 한다.

## 성공 기준

- 앱 가입자 목록의 직접 지급 버튼 문구가 추첨권 지급으로 보인다.
- 버튼 클릭 시 Supabase의 추첨권 번호 재활용 정책을 그대로 사용한다.
- 지급 결과는 `raffle_tickets`, `events`, 유저 요약 집계에 반영된다.
- 추첨권 발급 제외 유저에게는 지급되지 않는다.

## 구현 범위

- 새 Supabase RPC `admin_grant_raffle_ticket(p_login_id, p_reason)`를 추가한다.
- 수동 지급은 매번 고유 `admin_manual_*` 조건으로 기록해 같은 유저에게 여러 번 지급할 수 있게 한다.
- admin 앱 가입자 탭의 `뽑기권 +1` 버튼을 `추첨권 +1`로 바꾼다.
- 기존 카드 뽑기권 직접 지급 RPC는 보존하되 admin UI에서는 더 이상 호출하지 않는다.

## 검증

- admin HTML 스크립트 문법을 확인한다.
- SQL 파일이 기존 `bu_issue_raffle_ticket` 흐름을 호출하는지 확인한다.
- 충돌 마커와 공백 오류를 확인한다.
