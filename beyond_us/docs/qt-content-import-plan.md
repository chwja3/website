# Q.T. 본문 게시 데이터화 계획

## 목표

`큰은혜 2026년 05월/06월` PDF를 앱에서 직접 읽지 않고, 날짜별 Q.T. 본문 데이터로 변환해 `QTContents` 시트에 올릴 수 있게 준비한다.

## 접근

1. PDF가 텍스트 추출 가능한 구조인지 확인한다.
2. 가능하면 날짜 단위로 본문을 나누어 JSON/CSV 초안을 만든다.
3. 날짜 자동 분리가 불안정하면 페이지별 PDF 또는 텍스트를 먼저 저장해 수동 검수 가능한 형태로 만든다.
4. 최종 앱 구현은 `QTContents` 시트를 읽는 방식으로 설계한다.

## 검증 기준

- 페이지 수와 날짜별 매칭 결과를 확인할 수 있다.
- 날짜별 추출 결과가 사람이 검수 가능한 파일로 남는다.
- 앱/GAS는 원본 PDF를 매번 읽지 않는 방향으로 유지한다.
