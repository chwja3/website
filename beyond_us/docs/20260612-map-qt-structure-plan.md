# 2026-06-12 맵/QT 구조 계획

## 목표

워십팀과 프로그램팀 자료가 오기 전에, 광범위수사 맵과 Q.T. 말씀 묵상 입력 기능이 들어갈 앱 구조를 먼저 만든다.

## 맵 구조

- 광범위수사 탭 안에 하위 탭을 둔다.
- 하위 탭은 `포스터`, `전체`, `갈릴리동`, `예루살렘동`, `부스` 순서로 둔다.
- 각 하위 탭은 독립 이미지 슬롯을 가진다.
- 이미지는 파일이 준비되면 WebP로 넣는 것을 기본으로 한다.
- 예상 파일 경로는 아래와 같다.
  - `images/maps/crime-poster.webp`
  - `images/maps/overview.webp`
  - `images/maps/galilee.webp`
  - `images/maps/jerusalem.webp`
  - `images/maps/booths.webp`
- 아직 파일이 없으면 준비 중 안내를 보여준다.

## QT 구조

- 본문 이미지는 기존 날짜별 이미지 로딩 구조를 유지한다.
- 사용자는 QT 페이지에서 `답변`과 `기도제목`을 입력할 수 있다.
- 저장 데이터는 `qt_submissions` 테이블에 쌓는다.
- 보상 정책은 추후 확정 전까지 지급하지 않는다.
- 이후 정책이 정해지면 제출 RPC에서 카드 뽑기권 또는 추첨권 지급을 연결한다.

## 비범위

- 맵 이미지 제작.
- QT 본문/질문 원문 작성.
- QT 완료 보상 지급 정책 확정.
- Admin QT 제출 현황 화면.
