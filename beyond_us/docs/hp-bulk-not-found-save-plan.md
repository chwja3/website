# H&P 유저 없음 일괄 저장 Plan

## 목적

H&P 기도제목 매칭 탭에서 `유저 없음` 상태의 항목을 운영자가 하나씩 저장하지 않아도 되도록 일괄 저장 버튼을 추가한다.

## 범위

1. H&P 기도제목 매칭 헤더에 `유저 없음 전체 저장` 버튼을 추가한다.
2. 버튼은 `matchState === not_found`이고 `ownerNameInput`과 `entryId`가 있는 항목만 대상으로 한다.
3. 기존 `admin_match_hold_pray_entry` RPC를 재사용해 별도 SQL 추가 없이 동작하게 한다.
4. 저장 중에는 버튼을 비활성화하고 진행 상태를 표시한다.
5. 완료 후 H&P 매칭 목록을 다시 불러온다.

## 검증 기준

- `유저 없음` 항목이 없으면 버튼이 안내만 띄우고 서버 호출을 하지 않는다.
- 대상이 있으면 각 항목을 기존 이름으로 저장한다.
- 저장 실패가 있으면 실패 개수를 안내한다.
