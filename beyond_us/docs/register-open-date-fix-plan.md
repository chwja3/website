# 신규 가입 후 티저 진입 수정 계획

## 목표

신규 가입 직후 앱 오픈일 정보가 비어 있어 티저/커밍순 화면으로 잘못 이동하는 문제를 막는다.

## 접근

1. GAS `registerUser()`가 로그인 응답과 같은 앱 진입 판단 정보를 반환하게 한다.
2. 프론트 회원가입 성공 분기에서 `data.appOpenDate`, `data.isStaff`, `data.isDev`, `data.parish`를 저장한다.
3. `shouldEnterApp()` 판단에 localStorage의 오래된 값이 아니라 방금 받은 서버 응답을 사용한다.
4. 프론트 버전을 함께 올려 캐시 갱신이 일어나게 한다.

## 성공 기준

- 신규 가입 직후 `appOpenDate`가 오늘 이전이면 바로 앱 화면으로 진입한다.
- 앱 오픈 전 사용자만 티저 화면을 본다.
- 기존 로그인 경로는 그대로 유지된다.
