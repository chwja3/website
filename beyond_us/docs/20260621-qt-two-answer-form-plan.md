# QT 답변 2개 폼 변경 계획

## 목표

QT 말씀 묵상 입력 폼에서 `답변 3`을 제거하고, 완료 조건을 `답변 1`, `답변 2`, `기도제목`으로 단순화한다.

## 범위

- 사용자 앱 QT 입력 UI에서 `답변 3` 필드를 제거한다.
- 프론트 저장/불러오기 로직은 답변 2개만 다룬다.
- Supabase `submit_qt_reflection_v2`는 기존 인자 호환성을 유지하되, 카드팩 보상 조건은 답변 2개와 기도제목 기준으로 바꾼다.
- 앱 캐시 버전을 갱신한다.

## 검증

- `app.html`에서 `qtAnswer3Input`이 사라진다.
- `app.js` 문법 검사를 통과한다.
- 새 SQL migration에서 `answerCount`와 `missionCleared` 기준이 2개 답변으로 바뀐다.
