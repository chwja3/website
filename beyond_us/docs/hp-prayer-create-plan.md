# H&P 새 기도제목 추가 계획

## 목표

관리자가 H&P 기도제목 매칭 탭에서 새로 들어온 기도제목과 작성자 정보를 직접 추가할 수 있게 한다.

## 범위

1. Supabase에 새 RPC `admin_create_hold_pray_entry`를 추가한다.
2. 입력값은 기도제목, 사람 이름, 선택 닉네임, 익명 여부, 사용자 화면 노출 여부로 둔다.
3. 새 기도제목은 기본적으로 `week_key = null`로 저장해 전체 H&P 기도제목 풀에 포함한다.
4. 닉네임이 입력되면 `profiles.login_id`로 우선 매칭한다.
5. 닉네임이 없으면 실명 기준으로 활성 유저가 정확히 1명일 때만 `profile_id`를 연결한다.
6. 동명이인이거나 유저를 찾지 못하면 `profile_id`는 비워 두고 `owner_name_input`에 입력한 이름을 남긴다.
7. Admin H&P 기도제목 매칭 탭 상단에 새 기도제목 추가 박스를 배치한다.
8. 저장 후 매칭 목록을 다시 불러와 동명이인 선택 또는 미매칭 후속 처리를 이어갈 수 있게 한다.

## 검증 기준

- 기도제목과 사람 이름을 입력해 새 행을 만들 수 있다.
- 매칭 성공 시 `hold_pray_entries.profile_id`가 연결된다.
- 동명이인 또는 미가입자는 기존 매칭 목록의 확인 필요 박스에 표시된다.
- 사용자 화면 노출 체크가 꺼진 경우 `visible = false`로 저장된다.
