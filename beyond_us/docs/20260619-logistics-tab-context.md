# 숙소/차량 배정표 탭 컨텍스트

## 결정 사항

- 실제 숙소/차량 배정표 파일 형식은 아직 이 대화에서 제공되지 않았다.
- 그래서 이번 작업은 데이터를 담을 표준 구조와 조회 UI를 먼저 만든다.
- 유저 개인 배정표는 개인정보 성격이 있으므로 서버에서 `auth.uid()` 기준으로 본인만 조회한다.
- 어드민은 전체 배정표를 확인해야 하므로 `bu_admin_profile()` 권한 확인을 사용한다.
- 새 탭은 `tab_settings`에 `logistics` 키로 넣고, 초기 상태는 오픈으로 둔다.

## 데이터 필드

- 기본 정보는 이름, 닉네임, 교구, 조 이름을 둔다.
- 숙소 정보는 건물, 방, 숙소 조, 비고를 둔다.
- 차량 정보는 차량 조, 노선, 차량 번호, 출발 정보, 좌석, 비고를 둔다.
- 예외적인 안내는 `raw_note`에 담는다.

## 남은 운영 작업

- 실제 배정표가 나오면 `retreat_logistics_assignments`에 import해야 한다.
- import 형식이 정해지면 별도 CSV/XLSX import 스크립트를 만들 수 있다.
