# 미보유 카드 지급권 Plan

## 목표

BBB MISSION 1, 2와 시크릿 버디 정답 보상은 일반 카드팩과 분리된 미보유 엔 카드 지급권으로 처리한다. 미보유 엔 카드 지급권은 개봉 시점 기준 미보유 일반 엔 카드 9종이 있으면 그중 하나가 나오고, 미보유 일반 엔 카드가 없으면 일반 카드팩처럼 1-9번 일반 엔 카드 중 랜덤으로 나온다. BBB MISSION 3, 즉 현장 천로역정 7곳 완료 보상은 카드팩이 아니라 레어 엔 카드 10번을 직접 지급한다.

## 이번 로컬 작업 범위

- 앱 사전미션 카드팩 영역에는 사용자 기준 카드팩 버튼을 하나만 노출한다.
- 사용자 화면에서는 일반/특별 카드팩 구분 표현을 숨기고 모두 “카드팩”으로 표시한다.
- 카드팩 버튼을 누르면 일반 카드팩 잔여 수가 먼저 소비되고, 일반 카드팩이 없을 때만 내부적으로 `drawSpecialCard` 액션을 호출한다.
- 사용 가능한 카드팩 수 표시는 `🎫` 배지에 일반 카드팩과 특별 카드팩 잔여 수를 합산하고, `💜` 배지에 그중 특별 카드팩 잔여 수만 별도로 표시한다.
- BBB MISSION 1, 2는 운영진 승인 시 미보유 엔 카드 지급권을 지급한다.
- 시크릿 버디 정답 보상도 미보유 엔 카드 지급권을 지급한다.
- BBB MISSION 3은 7곳 사진을 모두 채운 시점에 레어 엔 카드 10번을 직접 지급한다.

## 서버 계약

- `getTabSettings`와 dashboard 응답의 `tabSettings.specialPack`이 `true`일 때만 특별 카드팩 잔여 수를 사용자 카드팩 총량에 합산한다.
- `userStatus.pendingSpecialPacks`가 특별 카드팩 잔여 수다.
- BBB MISSION 1, 2 승인 보상과 시크릿 버디 정답 보상은 내부적으로 `special_pack.granted` 이벤트로 기록한다.
- BBB MISSION 3 완료 보상은 `card.granted` 이벤트로 레어 엔 카드 10번을 직접 기록한다.
- 프론트엔드는 `pendingDraws > 0`이면 `drawCard`, `pendingDraws === 0 && pendingSpecialPacks > 0`이면 `drawSpecialCard`를 호출한다.
- `drawSpecialCard`는 개봉 시점의 최신 Collection 기준으로 일반 9종 중 미보유 카드가 있으면 미보유 카드만 후보로 잡는다.
- 미보유 일반 카드가 없으면 `no_missing_cards`를 반환하지 않고 일반 카드팩처럼 1-9번 일반 카드 중 랜덤으로 지급한다.

## 확정 정책

- 모든 일반 카드를 보유한 유저의 특별 카드팩은 일반 카드팩처럼 랜덤 지급한다.
- 레어 엔 카드는 카드팩 랜덤 후보에 포함하지 않고 현장 천로역정 완료 보상으로만 지급한다.
