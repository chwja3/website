# 추첨권 구현 계획

## 목표

카드 컬렉션 탭에서 사용자가 본인의 추첨권 개수를 바로 확인하고, 추첨권 배지를 누르면 전체 추첨권과 내 추첨권이 시각적으로 강조된 추첨통 모달을 볼 수 있게 한다.

## 구현 범위

1. GAS `userStatus` 응답의 `raffle` 계산 정책을 확정 정책으로 교체한다.
2. 자동 추첨권은 가입 1장, 카드 3종 1장, 카드 5종 1장, 카드 10종 1장으로 계산한다.
3. 수련회 참석 1장은 admin에서 체크하는 `RetreatAttendance` 시트로 관리한다.
4. admin 운영 메뉴에 수련회 참석자 관리 탭을 추가한다.
5. 컬렉션 상단 문구 옆 추첨권 배지와 아크릴 추첨통 모달은 새 계산값을 그대로 사용한다.
6. 프론트 변경에 맞춰 `APP_VERSION`, `version.txt`, `sw.js` 캐시명을 동기화한다.

## 검증 기준

- `userStatus` JSON에 새 정책 기준 `raffle.myTickets`, `raffle.totalTickets`, `raffle.fillPercent`가 포함된다.
- admin 수련회 참석자 관리 탭에서 교구, 이름, 닉네임, 수련회 참석 체크를 볼 수 있다.
- admin 체크 변경 후 해당 유저의 추첨권에 참석 1장이 반영된다.
- 컬렉션 탭에서 `n/10종 보유` 옆에 추첨권 배지가 보인다.
- 추첨권 배지를 누르면 모달이 열리고 닫힌다.
- 전체 추첨권이 1000장 이상이면 통 채움이 90%로 고정된다.
- 내 추첨권 수만큼 황금색 추첨권이 더미 위에 표시된다.
