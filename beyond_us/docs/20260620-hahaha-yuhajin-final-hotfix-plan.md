# 하하하 유하진 매칭 최종 보정 계획

## 목표

닉네임 `하하하` 앱 가입자를 조 명단의 `유하진`으로 확실히 연결하고, H&P 비익명 카드 정답 정책을 입력값 존재 기준으로 다시 고정한다.

## 범위

- `login_id` 또는 `display_name`이 `하하하`인 active 프로필을 정확히 1명 찾아 `name = 유하진`으로 보정한다.
- 4조 `유하진` 조 명단 row를 정확히 1개 찾아 해당 프로필에 `matched_manual`로 연결한다.
- 같은 프로필이 다른 조 명단 row에 잘못 연결되어 있으면 해제한다.
- `group_members`와 B.B.B. assignment 집계를 다시 동기화한다.
- H&P 제출, 조회, 재계산 정책을 비익명 카드의 비어 있지 않은 입력 기준으로 재적용한다.

## 검증

- SQL 파일에 충돌 마커가 없어야 한다.
- 하하하, 유하진, H&P 함수 키워드가 포함되어야 한다.
- 공백 검사를 통과해야 한다.
