# Admin post-GAS fixes 계획

Supabase 전환 후 admin 화면에서 발견된 세 가지 운영 버그를 고친다.

1. 추첨권 번호 탭의 회수된 번호 목록이 길어도 잘리지 않고 확인 가능하게 한다.
2. 유저 비밀번호 초기화 목록을 앱 가입자 탭처럼 소속 교구와 이름 기준으로 정렬한다.
3. 시스템 상태의 `admin_audit_user_state` 실패 원인을 제거하고, Supabase 상태 점검 결과가 정확히 표시되게 한다.

성공 기준.

- 회수된 추첨권 번호가 여러 개여도 화면 안에서 줄바꿈 또는 스크롤로 끝까지 보인다.
- 비밀번호 초기화 목록이 교구 순서, 이름 순서로 정렬된다.
- 시스템 상태에서 audit 점검이 정상 RPC를 호출하고 실패 시 원인을 더 명확히 보여준다.
