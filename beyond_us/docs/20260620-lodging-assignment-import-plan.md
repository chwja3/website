# 숙소 배정표 import 계획

## 목표

`숙소배정안내.pdf`의 숙소 배정표를 Supabase `retreat_logistics_assignments`에 넣고, 사람 이름과 교구를 기준으로 앱 가입자 닉네임을 매칭한다.

## 범위

- PDF에서 연번, 이름, 교구, 숙소 동, 호수를 추출한다.
- `1교구`와 `1청`처럼 표기가 다른 교구를 같은 기준으로 정규화한다.
- 이름과 정규화된 교구가 정확히 하나의 활성 프로필에 매칭되면 `profile_id`, `login_id`를 채운다.
- 매칭 후보가 없으면 앱 미연결 상태로 남긴다.
- 동일 이름과 동일 교구에 후보가 여러 명이면 자동 매칭하지 않고 비고에 후보 닉네임을 남긴다.
- `*`가 붙은 호수는 방장으로 `lodging_note`에 표시한다.

## 제외

- 차량 배정은 이 PDF에 없으므로 이번 import에서는 비워둔다.
- 중복 이름 수동 선택 UI는 이번 요청 범위에 포함하지 않는다.

## 검증

- PDF 추출 결과가 217명인지 확인한다.
- SQL 파일에 217개 row가 들어가는지 확인한다.
- `node --check beyond_us/app.js`와 어드민 inline script parse를 확인한다.
- `git diff --check`와 충돌 마커 검색을 실행한다.
