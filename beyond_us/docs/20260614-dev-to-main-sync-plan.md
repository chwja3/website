# 2026-06-14 dev to main 동기화 계획

## 목표

dev에는 반영됐지만 main에는 아직 없는 기능을 main에 반영하고, main Supabase에 실행할 SQL을 하나로 묶는다.

## 성공 기준

- main-only 수정인 교환 신청 최신 상태 재검증과 카드 뽑기권 조건 완화가 보존된다.
- dev-only 기능인 광범위수사, 목사님께 무물, 별빛 우편함, 맵/QT 구조, 수동 보상, QT 답변 및 완료 보상이 main에 들어간다.
- Supabase에 추가 실행해야 하는 migration SQL이 하나의 파일로 제공된다.
- 앱 JS 문법 검사와 충돌 표식 검사를 통과한다.

## 구현 방향

- `main` 브랜치에서 `dev`를 병합한다.
- 충돌이 나면 main의 최신 안정성 수정과 dev의 신규 기능을 모두 살린다.
- `main..dev` 기준 신규 migration을 순서대로 통합 SQL 파일에 합친다.
- 검증 후 의미 있는 커밋으로 남긴다.
