# 카드 보유 수량과 실물 수령 탭 불일치 조사 계획

컬렉션 화면과 admin 실물 카드 수령 탭이 같은 사용자 카드 보유 정본을 기준으로 숫자를 표시하는지 확인한다.

## 확인 기준

- 사용자 컬렉션은 Supabase `user_cards` 기준이어야 한다.
- admin 실물 카드 수령 탭의 `보유` 수량도 같은 `user_cards.quantity`를 보여야 한다.
- 실물 수령 탭의 `수령`, `미수령`, `TF 배부 필요` 같은 운영 보조 수량은 별도 계산이어도 된다.

## 작업 순서

1. admin 카드 수령 UI가 어떤 필드를 표시하는지 확인한다.
2. `admin_card_stats()` RPC가 어떤 필드를 내려주는지 확인한다.
3. 불일치 원인이 총 보유 수량, 실물 수령 수량, TF 배부 대상 수량 중 어디인지 분리한다.
4. 필요하면 표시 필드와 서버 응답을 정본 기준으로 보정한다.
