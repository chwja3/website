# 하하하 유하진 매칭 최종 보정 컨텍스트

## 확인한 문제

- 앱 가입자는 닉네임 `하하하`, 실명 `하진`으로 들어가 있었다.
- 조 명단에는 4조 `유하진`으로 들어가 있어서 이름 기준 자동 매칭과 수동 매칭 RPC가 막힐 수 있다.
- 운영 정책상 H&P 비익명 카드는 어떤 이름이든 입력하면 정답 처리되어야 하는데, 일부 함수가 이름 비교를 계속 쓰면 사용자별 결과가 달라질 수 있다.

## 결정

- 이번 보정은 이름/교구 추정 대신 `login_id` 또는 `display_name`의 `하하하`를 직접 기준으로 한다.
- 동일 프로필이 다른 조 명단 row에 연결되어 있으면 `manual_unmatched`로 해제하고, 4조 `유하진` row에 `matched_manual`로 고정한다.
- H&P 함수는 같은 migration 안에서 다시 정의해, 이 파일만 실행해도 정책이 적용되도록 한다.
