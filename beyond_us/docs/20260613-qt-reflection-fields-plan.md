# 2026-06-13 QT 답변 입력칸 정책 계획

## 목표

Q.T. 말씀 묵상 탭에서 2026년 6월 20일과 2026년 6월 21일에만 질문 답변 3개와 기도제목을 입력할 수 있게 한다.

## 성공 기준

- 2026년 6월 19일까지의 QT는 날짜별 QT 이미지만 보인다.
- 2026년 6월 20일 QT는 답변 1, 답변 2, 답변 3, 기도제목 입력칸이 보인다.
- 2026년 6월 21일 QT도 같은 입력칸이 보인다.
- 답변 입력칸은 기존 답변칸보다 얇게 보인다.
- 기존 QT 이미지 로딩 방식은 유지한다.

## 구현 범위

- 앱 QT HTML을 답변 3개와 기도제목 구조로 바꾼다.
- 클라이언트에서 입력 가능 날짜를 `2026-06-20`, `2026-06-21`로 제한한다.
- Supabase `qt_submissions`에 답변 2, 답변 3 컬럼을 추가한다.
- 새 RPC `submit_qt_reflection_v2`로 답변 3개와 기도제목을 저장한다.
- 기존 `get_qt_reflection`은 답변 배열을 함께 내려주도록 갱신한다.

## 검증

- `app.js` 문법 검사를 통과한다.
- HTML 안의 QT 입력 id와 JS 참조가 일치한다.
- 충돌 마커와 공백 오류를 확인한다.
