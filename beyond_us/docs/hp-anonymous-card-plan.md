# H&P 익명 기도카드 처리 계획

## 목표

H&P 익명 기도카드는 사용자에게 익명 제출 카드로만 보여주고, 이름 맞추기 입력칸과 힌트 요청을 제공하지 않는다. 동시에 사용자에게 배정되는 3장의 H&P 카드 중 가능한 경우 최소 1장은 비익명 카드가 되게 해 3주차와 6주차 뽑기권 획득 기회를 보장한다.

## 범위

1. Supabase `bu_hold_pray_cards_for_profile` helper를 수정한다.
2. 카드 후보 중 비익명 카드가 있으면 먼저 비익명 카드 1장을 고정으로 선택한다.
3. 나머지 2장은 기존처럼 사용자와 주차 기준의 안정적인 정렬값으로 채운다.
4. 후보에 비익명 카드가 전혀 없으면 기존처럼 익명 카드만이라도 최대 3장 보여준다.
5. `submit_hold_pray_guess`에서 익명 카드에 대한 정답 제출을 서버에서도 거절한다.
6. 앱 UI의 익명 카드 문구를 “익명으로 제출된 기도카드예요.”로 맞춘다.

## 검증 기준

- 익명 카드에는 이름 입력칸과 힌트 요청 버튼이 보이지 않는다.
- 비익명 후보가 하나 이상 있으면 `bu_hold_pray_cards_for_profile` 결과 3장 안에 비익명 카드가 포함된다.
- 익명 카드에 정답 제출 RPC가 호출되어도 `{ ok:false, error:'anonymous' }`로 거절된다.
- 앱 버전과 `version.txt`가 동일하게 갱신된다.
