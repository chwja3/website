# 성능 최적화 2차 계획

## 목표

초기 진입, 카드팩 상태 반영, H&P/B.B.B 탭 진입에서 발생하는 불필요한 서버 호출과 무거운 Sheet 읽기를 줄인다.

## 범위

1. 기본 사용자 상태 조회를 가벼운 `userStatusLite`로 분리한다.
2. 추첨권 상세 통계는 필요한 시점 또는 백그라운드에서 전체 `userStatus`로 보강한다.
3. 로그인 직후 `loadTrades`, H&P, B.B.B 예열이 한 번에 몰리지 않도록 지연한다.
4. B.B.B 기본 데이터와 메시지 데이터를 분리해 화면 표시를 먼저 끝낸다.
5. 앱 버전과 서비스워커 캐시 버전을 동기화한다.
6. 배포 시 `images/unused`를 제외할 수 있는 정적 설정을 추가한다.

## 제외

- Sheet 구조를 새로 만드는 대시보드 사전계산화는 별도 단계로 둔다.
- 기존 관리자 수동 실행 함수와 마이그레이션 함수는 삭제하지 않는다.
- 추첨권 정책과 카드 지급 정책은 변경하지 않는다.

## 검증 기준

- 로그인 직후 기본 화면이 기존보다 적은 API로 렌더링된다.
- Collection과 카드팩 버튼은 `userStatusLite`만으로 정상 표시된다.
- 추첨권 모달을 열면 전체 추첨권 정보가 보강된다.
- B.B.B 탭은 메시지 로드를 기다리지 않고 기본 미션 UI를 먼저 표시한다.
- `node --check beyond_us/app.js`, `node --check beyond_us/sw.js`, `git diff --check`가 통과한다.
