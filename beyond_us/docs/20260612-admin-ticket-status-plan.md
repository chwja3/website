# 2026-06-12 관리자 뽑기권/추첨권 현황 계획

## 목표

앱 가입자 탭의 `누락 보상` 도구를 `뽑기권/추첨권 현황`으로 바꾸고, 같은 화면에서 추첨권 수동 추가와 활성 추첨권 회수를 처리한다.

## 성공 기준

- 앱 가입자 행의 버튼 문구가 `뽑기권/추첨권 현황`으로 보인다.
- 현황 카드에서 카드 뽑기권, 추첨권, 보유 카드 종수를 한 번에 확인할 수 있다.
- 현황 카드에서 추첨권 1장을 추가할 수 있다.
- 활성 추첨권 목록에서 번호별로 회수할 수 있다.
- 회수된 추첨권 번호는 비활성 상태가 되어 이후 발급에 재사용될 수 있다.

## 구현 범위

- 새 RPC `admin_get_profile_raffle_tickets(p_login_id)`를 추가한다.
- 새 RPC `admin_revoke_raffle_ticket(p_login_id, p_ticket_no, p_reason)`를 추가한다.
- 기존 조건형 보상 지급 RPC는 유지한다.
- admin UI의 `누락 보상` 문구와 직접 추첨권 지급 위치를 현황 카드 안으로 옮긴다.

## 검증

- admin HTML inline script 문법을 확인한다.
- 충돌 마커와 공백 오류를 확인한다.
- 새 SQL에 schema reload 알림이 포함되어 있는지 확인한다.
