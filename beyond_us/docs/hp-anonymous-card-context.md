# H&P 익명 기도카드 처리 컨텍스트

2026-05-20. 운영팀 논의에서 익명 기도제목은 사용자가 맞추는 대상이 아니며, “익명으로 제출된 기도카드예요.”처럼 안내만 보여주기로 했다.

앱 UI에는 이미 `card.anon`일 때 이름 입력칸을 숨기는 분기가 있었다. 다만 서버의 H&P 카드 선택 helper는 3장을 단순히 안정 정렬로 뽑고 있어, 사용자에게 익명 카드 3장만 배정될 수도 있었다. 3주차와 6주차에는 H&P 정답을 맞춰야 뽑기권을 받을 수 있으므로 가능한 경우 최소 1장은 비익명 카드가 포함되어야 한다.

서버 방어도 필요하다. 프론트에서 입력칸을 숨겨도 RPC를 직접 호출할 수 있으므로, `submit_hold_pray_guess`는 익명 카드에 대해 `anonymous` 오류를 반환해야 한다.

카드 선택은 기존 deterministic 성격을 유지한다. 사용자 id, 주차, 기도제목 id를 섞은 `md5` 정렬값을 계속 쓰되, 비익명 후보가 있으면 가장 앞의 비익명 1장을 먼저 고정하고, 나머지는 동일 정렬값으로 채운다.
