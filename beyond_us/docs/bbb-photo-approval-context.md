# BBB 사진 승인제 Context

## 배경

BBB 미션 1, 2는 사진이 미션 조건을 만족하는지 사람이 확인해야 한다. 기존 구현은 사용자가 사진을 업로드하면 서버에서 바로 `ticket.granted` 이벤트를 생성했다. 이 방식은 운영자가 사진을 확인하기 전에 뽑기권이 지급되므로 공정성 관리가 어렵다.

## 결정

- `m1`, `m2`만 승인제로 전환한다.
- `m3_0`부터 `m3_6`까지의 천로역정 맵 사진은 기존처럼 7개 스팟 완료 시 자동 보상한다.
- 새 시트를 만들지 않고 기존 `BBBPhotos`에 승인 상태 컬럼을 추가한다.
- 승인 시 보상은 기존 이벤트 원장인 `Events`에 `ticket.granted`로 남긴다.
- 중복 지급 방지는 기존 `legacySource` 기준을 유지한다. `m1`은 `bbb_photo`, `m2`는 `bbb_m2`를 쓴다.

## 운영 흐름

1. 사용자가 앱에서 m1 또는 m2 사진을 업로드한다.
2. 앱에는 운영진 확인 대기 상태가 표시된다.
3. 관리자가 admin의 `BBB 사진 승인` 탭에서 사진을 확인한다.
4. 관리자가 승인하면 티켓 지급 이벤트가 생성되고 `Collection`이 재계산된다.
5. 이미 지급된 사진은 중복 승인해도 추가 티켓이 지급되지 않는다.

## 추가 UX 결정

- admin 승인 화면은 사진 제출 건 단위가 아니라 유저 단위로 묶는다.
- 한 유저 행 안에 `MISSION 1`과 `MISSION 2`를 가로로 배치해 줄내림을 줄인다.
- 실물 카드 수령 탭처럼 닉네임 또는 이름으로 검색할 수 있게 한다.
- 승인 외에 거절을 제공한다. 거절된 사진은 티켓을 지급하지 않고 `approvalStatus = rejected`로 표시한다.
- 사용자는 거절된 사진을 삭제하거나 새 사진으로 덮어써서 다시 제출할 수 있다. 새로 올리면 상태는 `pending`으로 돌아간다.
