# 2026-06-12 맵/QT 구조 컨텍스트

## 배경

- 워십팀 요청 항목은 맵과 QT다.
- 맵은 이미지로 전달될 예정이다.
- QT는 말씀 본문, 질문, 답변, 기도제목 구조가 필요하다.
- 현재 앱에는 Q.T. 날짜별 이미지 표시와 광범위수사 포스터 표시만 있다.

## 결정

- 맵은 별도 앱 탭을 새로 만들지 않고 기존 `광범위수사` 탭 안에 둔다.
- 맵 이미지는 하위 탭별로 분리해서 받는 것을 표준으로 한다.
- QT는 본문 이미지를 그대로 보여주고, 답변/기도제목은 앱 입력 데이터로 저장한다.
- QT 보상은 아직 연결하지 않고, 추후 정책 확정 후 붙인다.

## 구현 메모

- 광범위수사 하위 탭은 `포스터`, `전체`, `갈릴리동`, `예루살렘동`, `부스`다.
- 맵 파일 경로는 `images/maps/*.webp`로 고정했다.
- 파일이 없으면 깨진 이미지 대신 준비 중 안내를 보여준다.
- 맵 이미지는 탭하면 기존 라이트박스로 확대해서 볼 수 있다.
- QT 제출 테이블은 `qt_submissions`다.
- 사용자 RPC는 `get_qt_reflection`, `submit_qt_reflection`이다.
- 제출 이벤트는 `events.event_type = qt.submitted`로 남긴다.
- 앱 캐시 버전은 `20260612i`로 올렸다.

- `git diff --check` 통과.
- 충돌 마커 검색 결과 없음.
