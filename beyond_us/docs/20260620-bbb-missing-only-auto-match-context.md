# BBB 미배정 자동 매칭 보강 컨텍스트

## 확인한 현재 구조

- `bbb_extra_care_roster_links`는 한 TF 조원이 여러 단독 부분참여자를 추가 케어할 수 있도록 만든 보조 테이블이다.
- 앱의 `get_bbb_status`는 `extraCareBuddies`를 내려주고, 유저 BBB 화면은 이 목록을 추가 케어 대상자로 표시한다.
- 기존 관리자 `자동 매칭` 버튼은 `admin_auto_assign_bbb_buddies`를 호출하며, 먼저 기존 매칭을 지운 뒤 다시 배정한다.

## 새 결정

- 운영 중 수동 보정된 매칭을 보호하기 위해 새 함수는 기존 매칭을 지우지 않는다.
- 일반 버킷에서는 케어버디가 비어 있는 사람만 후보를 찾고, 시크릿버디가 비어 있는 사람만 역방향 링크를 채운다.
- 토참/일참 단독 버킷에서는 비어 있는 경우에만 전참 케어버디와 TF 시크릿버디를 넣는다.
- TF 추가 케어 링크는 중복 없이 upsert해서 같은 TF가 여러 명을 볼 수 있게 유지한다.
