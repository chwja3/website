# 인증/문의 보안 패치 Context

## 배경

2026-05-13 사용자 제보로 다음 리스크를 확인했다.

- `Users` 시트의 `password` 컬럼에 유저 비밀번호가 평문으로 저장된다.
- `getInquiries` GET 액션이 `adminPw` 없이 전체 개발자 문의를 반환한다.
- 앱 HTML/JS에 GAS URL이 노출되는 것은 정적 프론트엔드 구조상 자연스러운 일이므로, 서버 액션별 권한 검증이 보안 경계가 되어야 한다.

## 결정

- 시트 구조를 크게 바꾸지 않고 기존 `password` 컬럼에 해시 문자열을 저장한다.
- 해시 포맷은 prefix로 구분해 legacy 평문과 공존시킨다.
- 로그인 성공 시 legacy 평문이면 즉시 해시로 업그레이드한다.
- 운영자가 수동 실행할 수 있도록 전체 평문 비밀번호 전환 dry-run/apply 함수를 제공한다.
- 개발자 문의 조회는 관리자 전체 조회 또는 본인 세션 조회만 허용한다.

## 운영 메모

- 기존 사용자는 해시 전환 후에도 같은 비밀번호로 로그인할 수 있다.
- 이미 발급된 세션 토큰은 유지한다. 전체 로그아웃이 필요하면 별도 세션 무효화 함수를 추가하는 편이 좋다.
- 이미 외부에 실제 평문 비밀번호가 유출되었다고 판단되는 경우, 해시 전환만으로는 과거 유출을 되돌릴 수 없으므로 비밀번호 초기화 공지가 필요하다.
