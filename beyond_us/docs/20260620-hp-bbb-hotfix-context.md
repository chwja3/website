# H&P와 B.B.B. 운영 핫픽스 컨텍스트

2026-06-20. 운영 중 H&P는 더 이상 실제 이름 매칭을 엄격히 보지 않고, 사용자가 이름을 입력하기만 하면 정답으로 처리하기로 했다.

B.B.B. Mission 1/2는 사진 승인 또는 성공 상태인데 카드팩이 생성되지 않는 사례가 있다. 보상 지급은 이벤트와 티켓 카운터가 중복 생성되지 않게 기존 이벤트 존재 여부를 보고 idempotent하게 보강한다.

케어버디 또는 시크릿버디가 앱에 가입하지 않은 조원일 수 있다. 이 경우 profile 정보가 없어도 `retreat_group_roster.participant_name`을 그대로 보여줘야 하므로, 조회 RPC와 프론트 표시 모두에서 `null` 대신 사람 이름 fallback을 사용한다.
