# 향후 기능 백로그 Plan

## 목적

수련회 현장 운영 전까지 남은 기능 아이디어를 한곳에 모아, 구현 가능한 단위와 기획 확인이 필요한 단위를 분리한다.

## 현재 완료로 분류한 항목

- 현장미션 MISSION 3 위치 정리. 사용자 기준 완료 항목으로 기록한다.
- Q.T. 말씀 묵상 페이지 준비중 화면.
- 구성 탭 레이아웃.
- 추첨권 번호 발급/표시/검색 시스템.
- 히든/레어 카드 미션 1차 정책. 천로역정 7스팟 완료 시 레어 엔 카드 직접 지급.
- 시크릿버디 정답 보상. 최초 정답 시 미보유 엔 카드 지급권 1장 지급.
- 카드팩 이펙트 강화. 80% 찢기 자동 오픈, 카드 등장, 십자 빔, 사운드/진동 싱크 조정.

## 남은 큰 작업 묶음

1. 천로역정 스팟별 QR 검증.
2. 레어카드 연출 구현.
3. 프로그램팀 요청 퀴즈.
4. 크라임씬/내 안의 빛 유형 검사.
5. BBB 조별 매칭.

## 진행 원칙

- 의미가 불분명한 항목은 바로 구현하지 않는다.
- 각 기능은 먼저 데이터 구조, 보상 정책, admin 운영 방식, 앱 UX를 확정한다.
- 현장 운영자가 확인해야 하는 기능은 admin 페이지 작업을 함께 설계한다.

## 2026-05-13 반영

- 추첨권 구현은 1차 마무리로 본다.
- 확인용 DEV 프리뷰는 유지하며, 당시 TODO에는 `추첨권 디테일 수정`만 남겼다.

## 2026-05-15 반영

- DEV 최신 기준으로 추첨권은 번호 발급/표시/admin 검색까지 완료로 본다.
- 히든/레어 카드 미션은 천로역정 7스팟 인증 완료 시 레어 엔 카드 직접 지급으로 1차 완료로 본다.
- 레어카드는 랜덤 카드팩 뽑기 대상이 아니라 현장 보상 직접 지급 대상이다.
- `추첨권 디테일 수정` 항목은 `추첨권 현장 운영 최종 점검`으로 대체한다.

## 2026-05-15 추가 반영

- 레어카드 연출은 카드팩 기본 연출과 별도로 남긴다.
- H&P는 코드 하드코딩을 줄이고 시트 또는 admin 기반 데이터 관리로 전환할 필요가 있다.
- Q.T. 페이지는 준비중 화면 이후 매일 본문을 게시하는 운영 구조가 필요하다.

## 2026-05-19 재정리

- 사용자가 남은 TODO를 6개로 다시 지정했다.
- 현재 남은 목록에서는 추첨권 현장 운영 점검, 레어 엔 카드 자산 확정, H&P 하드코딩 제거를 제외한다.
- 천로역정 2스팟은 이전 적용 확인과 별개로 실제 현장미션 랜덤 미션 구현 TODO로 다시 추적한다.

## 2026-05-21 재정리

- Q.T. 매일 본문 게시는 날짜별 PNG 자동 표시와 Supabase Storage 우선 로딩으로 1차 완료 처리한다.
- 천로역정 유저별 2스팟 랜덤 배정과 빨간 원 표시는 완료 처리한다.
- 천로역정의 남은 핵심 작업은 스팟별 QR 검증이다. 현재는 배정 스팟에 사진을 올리면 완료 처리되므로, QR 값이 맞을 때만 완료되도록 보강해야 한다.
- admin 로그아웃 버튼은 완료 처리한다.
