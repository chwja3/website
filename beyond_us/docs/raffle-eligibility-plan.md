# 추첨권 대상 교구 제한 Plan

## 목표

추첨권 발급 여부를 admin의 앱 가입자 탭에서 직접 관리한다. 발급 제외로 체크된 사용자는 현재 추첨권을 모두 회수하고, 향후 조건을 충족해도 추첨권을 받지 않으며, 앱에서 추첨권 현황을 보지 않는다.

## 작업 범위

- `Users` 시트에 `raffleExcluded` 컬럼을 추가한다.
- admin 앱 가입자 탭에 추첨권 발급 제외 체크박스를 추가한다.
- 목양교구/교회학교 등 기본 대상이 아닌 교구는 `raffleExcluded`가 처음부터 체크된 상태로 생성되게 한다.
- 기존 가입자 중 기본 대상이 아닌 교구도 같은 기준으로 체크하고, 활성 추첨권을 회수한다.
- 추첨권 번호는 회수 후 재활용한다.
- `RaffleTickets`는 번호, 활성 여부, 받은 사람, 발급 조건을 보여주는 번호 풀 시트로 사용한다.
- admin 운영 쪽에 추첨권 번호 탭을 두고 현재 발급된 추첨권 총 장수와 번호 목록을 보여준다.
- admin 추첨권 번호 검색 줄은 작은 화면에서도 버튼이 박스 밖으로 밀리지 않게 고정한다.
- admin 추첨권 번호 탭의 `사용 가능 번호` 표현은 실제 의미에 맞춰 `회수된 번호`로 바꾼다.
- DEV 개발자 초기화는 카드, 뽑기권 이벤트, 활성 추첨권 번호를 함께 초기화한다.
- 추첨권 회수 후 카드 3종, 5종, 10종 조건은 다시 달성하면 새 번호를 받을 수 있게 한다.

## 제외 범위

- 추첨 당첨 처리나 실제 추첨 로직은 이번 범위에 포함하지 않는다.
- admin 참석자 체크 UI는 앱 가입자 탭 안에 유지한다. 현재 참석 체크는 추첨권에 반영되지 않는다.
- DEV 초기화는 앱 가입 상태 자체를 삭제하지 않는다.
