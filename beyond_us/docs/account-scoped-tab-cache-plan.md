# 계정별 탭 캐시 버그 수정 계획

## 목표

Hold & Pray, B.B.B. 미션, 천로역정 탭에서 계정 전환 후 이전 계정 데이터가 보이지 않게 한다. 동시에 로그인 후 숨은 탭 데이터를 미리 불러와 첫 진입 체감 로딩을 줄인다.

## 접근

1. 로그인 계정이 바뀌거나 로그아웃할 때 계정별 메모리 상태와 탭 DOM을 초기화한다.
2. H&P와 BBB 로더가 현재 데이터의 계정 키를 보관하게 하고, 다른 계정이면 기존 로드 완료 상태를 사용하지 않는다.
3. 비동기 요청이 늦게 돌아와도 현재 계정과 요청 계정이 다르면 DOM에 반영하지 않는다.
4. 앱 진입 후 초기 데이터 동기화가 끝나면 H&P와 BBB 데이터를 지연 prefetch한다.
5. 프론트 캐시 버전을 함께 올린다.
