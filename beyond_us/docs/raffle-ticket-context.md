# 추첨권 구현 Context

## 결정 사항

- 추첨권은 카드팩 티켓과 별도 개념이다.
- 카드 컬렉션 탭의 `n/10종 보유` 문구 옆에 추첨권 배지를 둔다.
- 추첨권 배지는 카드팩 티켓과 다른 `🎟️` 느낌으로 표시한다.
- 배지를 누르면 전체 추첨권 수, 내 추첨권 수, 추첨통 시각화를 보여주는 모달을 연다.
- 추첨통 채움 정도는 전체 추첨권 1000장을 90% 상한으로 계산한다.
- 내 추첨권은 더미 맨 위에 황금색으로 빛나는 추첨권으로 강조한다.

## 확정 계산 정책

- 앱 가입 시 자동 추첨권 1장.
- 카드 3종 보유 시 자동 추첨권 1장 추가.
- 카드 5종 보유 시 자동 추첨권 1장 추가.
- 카드 10종 보유 시 자동 추첨권 1장 추가.
- 수련회 참석 시 추첨권 1장 추가.
- 수련회 참석 여부는 admin 페이지의 수련회 참석자 관리 탭에서 수동 체크한다.

## 데이터 흐름

- GAS는 `Users`를 기준으로 가입 추첨권과 전체 유저 목록을 계산한다.
- GAS는 `Collection` projection cache를 기준으로 각 유저의 카드 보유 종류를 읽는다.
- GAS는 `RetreatAttendance` 시트를 기준으로 참석 추첨권을 더한다.
- `getUserStatus()`가 현재 유저의 `raffle` 통계를 내려준다.
- 프론트는 기존 `userStatus` 캐시와 렌더 흐름을 그대로 사용한다.
- 참석 여부 저장을 위해 `RetreatAttendance` 시트를 사용한다.
