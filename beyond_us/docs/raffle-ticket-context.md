# 추첨권 구현 Context

## 결정 사항

- 추첨권은 카드팩 티켓과 별도 개념이다.
- 카드 컬렉션 탭의 `n/10종 보유` 문구 옆에 추첨권 배지를 둔다.
- 추첨권 배지는 카드팩 티켓과 다른 `🎟️` 느낌으로 표시한다.
- 배지를 누르면 전체 추첨권 수, 내 추첨권 수, 추첨통 시각화를 보여주는 모달을 연다.
- 추첨통 채움 정도는 전체 추첨권 1000장을 90% 상한으로 계산한다.
- 내 추첨권은 더미 맨 위에 황금색으로 빛나는 추첨권으로 강조한다.
- 사용자 앱에는 1000장/90% 같은 내부 시각화 규칙을 문구로 노출하지 않는다.
- 모달 설명 문구는 `수련회에서 맥북 네오를 받을 수 있는 기회!`로 둔다.
- 추첨권 획득 조건을 모달 하단에 보여주고, 달성한 조건에는 초록색 체크 도장을 표시한다.
- 내 추첨권 외의 추첨권은 회색으로 표시하고, 내 추첨권은 황금색 반짝임으로 더 강하게 강조한다.

## 확정 계산 정책

- 앱 가입 시 자동 추첨권 1장.
- 카드 3종 보유 시 자동 추첨권 1장 추가.
- 카드 5종 보유 시 자동 추첨권 1장 추가.
- 카드 10종 보유 시 자동 추첨권 1장 추가.
- 수련회 참석은 추첨권 조건에서 제외한다.
- 추첨권은 최대 4장이다.
- 수련회 참석 여부는 admin 페이지의 수련회 참석자 관리 탭에서 기록용으로 수동 체크한다.

## 데이터 흐름

- GAS는 `Users`를 기준으로 가입 추첨권과 전체 유저 목록을 계산한다.
- GAS는 `Collection` projection cache를 기준으로 각 유저의 카드 보유 종류를 읽는다.
- GAS는 `RetreatAttendance` 체크를 추첨권 계산에 더하지 않는다.
- `getUserStatus()`가 현재 유저의 `raffle` 통계를 내려준다.
- 프론트는 기존 `userStatus` 캐시와 렌더 흐름을 그대로 사용한다.
- 참석 여부 저장을 위해 `RetreatAttendance` 시트를 사용한다.

## DEV 프리뷰 메모

- DEV 환경에서 `?rafflePreview=1`을 붙이면 실제 GAS 데이터 대신 내 추첨권 4장, 누적 추첨권 1000장, 통 채움 90% 상태로 추첨권 모달을 확인한다.
- 이 프리뷰는 화면 확인용 임시 코드이며 PROD 호스트에서는 동작하지 않는다.
- 확인이 끝나면 `RAFFLE_PREVIEW_MODE`와 `getRaffleStatsFromStatus()`의 프리뷰 분기를 제거한다.
