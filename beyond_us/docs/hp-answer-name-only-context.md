# H&P 정답 실명 기준 고정 컨텍스트

## 결정

- H&P 정답은 `profiles.name`만 기준으로 판정한다.
- `profiles.login_id`와 `profiles.display_name`은 정답 후보에서 제외한다.
- 사용자가 이름 사이에 공백을 넣는 경우를 줄이기 위해 비교 시 앞뒤 공백과 내부 공백은 제거한다.
- 힌트는 이미 추가된 `profiles.name_initials`와 `bu_korean_initials(name)` 기반 초성 흐름을 유지한다.

## 이유

main에서 H&P 정답이 맞지 않는다는 보고가 있었고, 운영 정책은 “정답은 이름 기준, 힌트는 초성”으로 확정되었다. 정답 후보를 아이디와 닉네임까지 허용하면 운영자가 보는 정답 여부와 사용자 안내가 어긋날 수 있어 실명 기준 하나로 고정한다.
