# 교환 신청 간헐 실패 점검 계획

## 목표

main 기준으로 카드 교환 신청이 가끔 전송되지 않는 원인을 확인하고, 사용자에게 보이는 실패 지점을 줄인다.

## 성공 기준

- 교환 신청 UI가 어떤 조건에서 버튼을 활성화하는지 확인한다.
- `request_trade` RPC가 어떤 조건에서 `ok:false`를 반환하는지 정리한다.
- 실제 간헐 실패로 보이는 케이스가 서버 거절, stale 데이터, 중복 요청, 권한 문제 중 어디에 가까운지 판단한다.
- 필요한 경우 최소 수정으로 사용자 메시지 또는 서버 검증을 보강한다.

## 점검 순서

1. 앱 교환 모달의 상태 흐름과 요청 payload를 확인한다.
2. Supabase `request_trade`와 수락 시 보유 카드 검증을 확인한다.
3. 관리자 교환 로그와 이벤트 로그에서 실패 확인 가능성을 확인한다.
4. 위험이 낮은 보강이 있으면 적용하고 문서에 남긴다.
