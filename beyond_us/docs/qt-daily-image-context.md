# Q.T. 날짜별 이미지 게시 컨텍스트

## 2026-05-21 요청

사용자가 `beyond_us/QT` 폴더에 날짜별 Q.T. 본문 PNG 파일을 올렸다. 파일명은 `260501.png`부터 `260630.png`까지 `YYMMDD.png` 형식이다.

## 결정

- 오늘 날짜의 Q.T.는 앱에서 한국 시간 기준으로 계산한다.
- 2026-05-21이면 `QT/260521.png`를 표시한다.
- 정적 배포 환경에서는 디렉터리 listing을 사용할 수 없으므로, 파일 존재 여부는 이미지 `load`/`error` 이벤트로 판단한다.
- 현재 단계에서는 Supabase 테이블이나 admin 입력 구조를 만들지 않고, 이미지 파일을 그대로 노출한다.
- 날짜별 자동 노출만 구현하고, 묵상 질문/메모/제출 저장은 후속 작업으로 남긴다.

## 구현

- `app.html`의 Q.T. 준비중 화면을 오늘 본문 이미지 화면으로 교체했다.
- `app.js`에서 `renderTodayQt()`를 추가했고, Q.T. 탭 진입 시 한국 시간 기준 `QT/YYMMDD.png`를 로드한다.
- 오늘 파일이 없으면 “오늘 Q.T.가 아직 준비되지 않았어요.” 안내를 보여준다.
- 프론트 버전은 `20260521a`로 동기화했다.

## 2026-05-21 Storage 전환 보강

- Q.T. 이미지는 이제 Supabase Storage `beyond-us-photos/QT/YYMMDD.png`를 우선 로드한다.
- Storage 파일이 없을 때만 로컬 `QT/YYMMDD.png`로 fallback한다.
- 프론트 버전은 `20260521b`로 동기화했다.
