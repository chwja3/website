# H&P 유저 없음 일괄 저장 Context

2026-05-26. H&P 정답 판정이 `owner_name_input`을 사용할 수 있게 바뀌면서, 프로필이 없는 기도제목도 이름만 맞으면 정답 처리할 수 있게 됐다.

관리자 화면에는 여전히 `유저 없음` 항목마다 저장 버튼이 보인다. 운영자가 이를 하나씩 눌러야 하는지 헷갈릴 수 있어, `유저 없음` 상태 항목을 기존 입력 이름 그대로 일괄 저장하는 버튼을 추가한다.

새 SQL RPC는 만들지 않는다. 이미 존재하는 `admin_match_hold_pray_entry`가 `ownerNameInput`을 보존하고 정답 재계산까지 수행하므로, 프론트에서 해당 RPC를 순차 호출하는 방식으로 충분하다.

구현은 관리자 H&P 기도제목 매칭 카드 상단에 `유저 없음 전체 저장` 버튼을 추가하는 방식으로 했다. 대상은 `unresolved` 목록 중 `matchState === 'not_found'`이고 `ownerNameInput`이 있는 항목만이다. `multiple`과 `needs_save`는 실제 유저 연결 가능성이 있으므로 일괄 처리에서 제외한다.
