# Admin BBB Matching Plan

## Goal

운영자가 admin에서 조별 참여자를 보면서 케어버디를 지정할 수 있게 한다. 각 사용자의 시크릿버디는 다른 사용자가 나를 케어버디로 지정한 관계에서 자동으로 계산한다.

## Scope

1. Supabase RPC로 BBB 매칭 현황을 조회한다.
2. Supabase RPC로 특정 유저의 케어버디를 저장하거나 해제한다.
3. admin `B.B.B. 미션` 탭의 기존 준비중 카드를 실제 매칭 관리 UI로 교체한다.
4. 저장 후 `BBB/천로역정` 운영 현황에서도 동일한 매칭이 보이게 한다.

## Design

- 관리자는 각 유저의 `케어버디`만 선택한다.
- A의 케어버디를 B로 저장하면 B의 시크릿버디가 A로 자동 저장된다.
- 기존에 A가 다른 사람을 케어버디로 보고 있었다면 그 사람의 시크릿버디 연결을 함께 해제한다.
- 자기 자신을 케어버디로 지정하는 것은 막는다.
- 조 정보가 있으면 조별로 묶어서 보여주고, 조가 없는 유저도 별도 그룹에 표시한다.
- 조 안에서는 `전참`, `토참`, `일참` 참여구분별로 다시 묶는다.
- 케어버디 후보는 같은 조와 같은 참여구분 안의 사람으로 제한한다. 참여구분이 비어 있으면 임시로 `전참` 그룹에 둔다.

## Verification

- admin 매칭 현황 RPC가 active 유저와 기존 매칭을 반환하는지 확인한다.
- 케어버디 저장 RPC가 양방향 필드를 일관되게 갱신하는지 확인한다.
- admin UI가 저장 후 다시 읽어 최신 상태를 보여주는지 확인한다.
