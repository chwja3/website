# H&P 기도제목 작성자 매칭 계획

## 목표

관리자가 H&P 기도제목 중 작성자와 연결되지 않은 항목을 유저 실명 기준으로 매칭할 수 있게 한다.

## 범위

1. `hold_pray_entries`에 관리자가 입력한 작성자 이름을 저장할 필드를 추가한다.
2. 작성자 미연결, 이름 입력했지만 유저 미발견, 유저와 연결 완료 상태를 반환하는 관리자 RPC를 추가한다.
3. 관리자 입력 이름이 유저 정보의 실명과 정확히 1명 매칭되면 `hold_pray_entries.profile_id`를 연결한다.
4. 이름이 비어 있으면 매칭을 해제하고, 유저가 없거나 동명이인이면 입력 이름만 저장해 운영자가 확인할 수 있게 한다.
5. Admin 페이지에 `H&P 기도 매칭` 탭을 추가하고 세 상태를 위에서부터 여닫기 박스로 보여준다.
6. 기도제목 작성자 매칭은 주차별 현황이 아니므로 전체 `hold_pray_entries`를 대상으로 한다.

## 검증 기준

- 작성자 없는 전체 기도제목이 첫 번째 박스에 표시된다.
- 존재하지 않는 이름을 저장하면 두 번째 박스에서 `!` 상태로 표시된다.
- 유저 실명과 1명 매칭되는 이름을 저장하면 세 번째 박스에 표시된다.
- 매칭 변경 후 H&P 정답 재계산이 실행된다.
