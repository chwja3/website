# 관리자 운영 상태 화면 개편 컨텍스트

2026-05-19. H&P 힌트가 사용자 버튼 클릭 즉시 초성 반환 방식으로 바뀌면서 관리자 문의에서 H&P 힌트 요청을 처리할 필요가 없어졌다. 개발자 문의 탭에서는 H&P 전용 박스를 제거하고 일반 문의만 관리한다.

2026-05-19. 운영자가 봐야 하는 H&P 상태는 문의가 아니라 유저별 진행 현황이다. 별도 H&P 운영 조회 탭에서 유저별 기도제목 작성 여부, 맞힌 카드 수, 각 카드 정답 여부를 보여주는 방향으로 정리한다.

2026-05-19. BBB 사진 승인 탭은 기존 Mission 1/2 사진 승인 기능을 유지하되, 같은 화면에서 케어버디/시크릿버디 매핑과 천로역정 배정 스팟 및 성공 여부까지 확인할 수 있게 확장한다.

2026-05-19. `admin_hold_pray_status` RPC를 추가했다. 현재 주차 또는 입력 주차 기준으로 각 유저에게 보이는 H&P 3장, 입력한 이름, 정답 여부, 본인 기도제목 작성 여부를 반환한다.

2026-05-19. `admin_bbb_pilgrim_status` RPC를 추가했다. 유저별 케어버디/시크릿버디, Mission 1/2 사진 승인 상태, 천로역정 배정 스팟, 성공 스팟, Mission 3 완료 여부를 반환한다.

2026-05-19. 개발자 문의 UI는 미답변과 답변 완료 박스로 분리했고, 각 문의는 기본 닫힘 `details` 요소로 렌더링한다. 레거시 `[H&P 힌트 요청]` 문의는 계속 숨겨서 일반 문의 목록에 섞이지 않게 한다.
