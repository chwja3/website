# Supabase PROD 안정화 핫픽스 계획

## 목표

H&P 탭의 Supabase RPC 서버 오류를 줄이고, 관리자 시스템 상태 audit에서 `user_summary`가 원천 데이터보다 과집계되는 문제를 보정한다.

## 범위

- H&P 3장 조회, 정답 제출, 초성 힌트 RPC를 안전한 버전으로 다시 배포한다.
- H&P 카드 선택 기준을 공통 helper 함수로 묶어 조회, 정답, 힌트가 같은 카드 인덱스를 보도록 한다.
- 사용자 요약과 추첨권 동기화 트리거를 트랜잭션 마지막에 실행되게 통일한다.
- 현재 활성 유저의 추첨권과 요약을 migration 적용 시점에 한 번 다시 계산한다.

## 검증 기준

- Supabase SQL Editor에서 `20260519000400_hp_and_summary_hotfix.sql`가 성공해야 한다.
- 앱 H&P 탭이 서버 연결 오류 없이 3장만 보여야 한다.
- 힌트 버튼을 눌렀을 때 초성이 즉시 보여야 한다.
- 관리자 시스템 상태 audit mismatch가 0이어야 한다.
