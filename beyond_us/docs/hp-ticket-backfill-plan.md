# H&P 정답 보상 누락 보정 Plan

## 목적

H&P에서 이미 정답을 맞췄거나, 작성자 매칭 보정 뒤 정답으로 재계산된 유저가 뽑기권을 못 받는 상황을 막는다.

## 범위

1. H&P 보상 지급 로직을 공통 helper로 분리한다.
2. `submit_hold_pray_guess`는 현재 제출 정답뿐 아니라 해당 주차의 누적 정답 여부를 확인해서 보상한다.
3. H&P 정답 재계산 함수는 correct 값을 고친 뒤 누락된 H&P 보상을 backfill한다.
4. 이미 H&P 보상을 받은 유저에게는 중복 지급하지 않는다.
5. 기존 `w3`, `w6` 보상 주차 정책은 유지한다.

## 검증 기준

- `hold_pray_guesses.correct = true`가 하나라도 있고 같은 주차 H&P `ticket.granted`가 없으면 1장을 지급한다.
- 같은 유저, 같은 주차에는 H&P 보상이 한 번만 지급된다.
- 익명 카드는 여전히 정답 제출 대상이 아니다.
