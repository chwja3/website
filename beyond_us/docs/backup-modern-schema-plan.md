# Backup Modern Schema Plan

## 목표

백업 파일이 현재 Events 중심 시트 구조를 따르도록 백업 사본을 정리한다.

## 구현 방향

1. 백업 파일명에 현재 백업 스키마 버전을 표시한다.
2. 원본 스프레드시트는 그대로 두고, 생성된 백업 사본에만 탭 순서, 색상, 숨김 상태를 적용한다.
3. 백업 사본에 현재 운영 필수 시트가 빠져 있으면 빈 시트와 현재 헤더를 만든다.
4. 백업 결과 JSON에 파일명, 파일 ID, 스키마 버전, 정규화 결과를 포함한다.

## 검증 기준

- `backupSpreadsheet()`가 `BeyondUs_events_v2_YYYYMMDD_HHmm` 형식의 파일을 만든다.
- 백업 사본에서 현재 운영 시트는 표시되고 legacy 시트는 숨김 처리된다.
- 백업 사본 탭 순서와 색상이 `FINAL_SHEET_TABS` 기준으로 정리된다.
- Apps Script 문법 검사가 통과한다.
