# 성능 최적화 3차 계획

## 목표

남은 병목인 admin 대시보드 집계, 추첨권 전체 스캔, 탭별 반복 조회, admin 대형 테이블 렌더링을 줄인다.

## 범위

1. 대시보드 결과를 짧게 캐시하고, 명시적인 admin 재계산 경로를 둔다.
2. 추첨권 전체 행 스캔 결과를 요약 캐시로 재사용한다.
3. H&P, B.B.B, 천로역정 탭은 계정별 TTL과 로컬 캐시를 더 명확히 적용한다.
4. admin 대형 테이블은 기본 렌더링 개수를 제한하고 검색 시 전체 결과를 보여준다.
5. 프론트 버전과 서비스워커 캐시 버전을 동기화한다.

## 제외

- 새로운 영구 통계 시트 생성은 이번 단계에서 하지 않는다.
- 추첨권 정책과 번호 발급 규칙은 바꾸지 않는다.
- H&P 문구/정답 데이터화는 별도 기능 작업으로 둔다.

## 검증 기준

- admin 대시보드가 기존 응답 구조를 유지한다.
- 추첨권 개수와 번호 표시가 기존과 같은 결과를 낸다.
- H&P, B.B.B, 천로역정은 계정 전환 시 이전 계정 데이터가 보이지 않는다.
- admin 목록은 검색 전 초기 DOM 렌더링량이 줄어든다.
- `node --check beyond_us/app.js`, `node --check beyond_us/sw.js`, `node --check beyond_us/Apps_Script`, `git diff --check`가 통과한다.
