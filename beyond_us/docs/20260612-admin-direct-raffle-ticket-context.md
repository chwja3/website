# 2026-06-12 관리자 추첨권 직접 지급 컨텍스트

## 결정

- 사용자가 기존 직접 지급 기능을 카드뽑기권이 아니라 추첨권으로 바꾸길 원했다.
- 추첨권은 번호 재활용, 발급 제외, 이벤트 로그, 유저 요약 갱신 정책이 이미 있으므로 기존 `bu_issue_raffle_ticket` 함수를 통과하게 한다.
- 수동 지급은 자동 조건과 달리 같은 유저에게 여러 장 지급될 수 있어야 하므로 매번 고유한 `admin_manual_*` 조건 key를 만든다.

## 주의점

- 이전에 만든 `admin_grant_card_pack_ticket` 함수는 남겨두지만 admin UI에서는 더 이상 호출하지 않는다.
- main 또는 PROD 적용 시 새 SQL `20260612000500_admin_direct_raffle_ticket_grant.sql`을 실행해야 한다.
- 기존 카드뽑기권 직접 지급 SQL을 아직 실행하지 않았다면, 이번 요구만 기준으로는 실행할 필요가 없다.
