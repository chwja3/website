# Supabase 추첨권 정책 연결 계획

## 목표

Supabase 전환 뒤에도 앱 가입, 카드 3종, 카드 5종, 카드 10종 보유 조건의 추첨권이 자동으로 발급되고, 발급 제외나 비활성화 상태에서는 회수되도록 만든다.

## 계획

1. 추첨권 발급, 회수, 요약 갱신을 공통 SQL 함수로 분리한다.
2. `profiles` 변경에 붙는 트리거로 앱 가입 조건과 발급 제외 상태를 동기화한다.
3. `user_cards` 변경에 붙는 트리거로 카드 3종, 5종, 10종 조건을 동기화한다.
4. 기존 이관 데이터는 `backfill_raffle_tickets()`로 한 번 보정한다.
5. 사용자 앱은 기존처럼 `get_user_status()`의 `raffle` 객체를 읽는다.

## 검증 기준

- 발급 제외가 아닌 활성 유저는 `app_signup` 추첨권을 가진다.
- 일반 카드 3종, 5종, 10종을 만족하면 각각 `card_3`, `card_5`, `card_10` 추첨권을 가진다.
- 같은 조건의 추첨권은 중복 발급되지 않는다.
- `raffle_excluded=true` 또는 `account_status <> active` 상태면 활성 추첨권이 0장이다.
- 회수된 번호는 다음 발급 시 낮은 번호부터 재사용된다.
