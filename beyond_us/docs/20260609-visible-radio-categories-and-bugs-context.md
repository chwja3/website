# 2026-06-09 보이는 라디오 카테고리와 운영 버그 수정 컨텍스트

- 첨부 기획안 제목은 `보이는 라디오 「별빛 우편함」 빛으로 전하는 마음`이다.
- 기획안 카테고리는 `우리 조 MVP`, `버디에게`, `감동의 순간`, `미안했어요`, `응원 한마디`, `익명 폭로(?) 칭찬` 6종이다.
- 사용자 플로우는 카테고리 선택, 대상자 입력 선택, 사연 작성, 익명 제출이다.
- 사연 권장 길이는 기획안 기준 20~500자이지만, 기존 앱은 1000자까지 열어두고 있었다. 이번 구현은 운영 유연성을 위해 DB 제한은 1000자를 유지하고 UI 안내는 20~500자 권장으로 표시한다.
- admin 기획 요구는 카테고리, 대상, 시간 표시와 카테고리별 실시간 확인이다. 후보, 보류, 제외 상태 변경은 진행자가 즉시 분류할 수 있도록 이번 구현에 포함하고, 드래그 순서 편집은 추후 필요 시 `pinned_order`로 확장한다.
- 적용 SQL은 `beyond_us/supabase/migrations/20260609000100_visible_radio_categories_trade_prayers.sql`이다.
- 기존 보이는 라디오 사연은 기본 카테고리 `우리 조 MVP`로 유지한다.
- 교환 기도제목은 `get_user_trades`에서 상대방의 최신 visible H&P 기도제목을 `otherPrayer`로 내려주도록 보강한다.
- admin 카드명 표시는 ID 5를 `네헤`, ID 6을 `더무`로 고정한다.
